#  Troubleshooting Logs Overview

This section of the repository acts as a running knowledge base for tracking errors, formatting issues, and other technical problems that come up during development.  
The goal is to document issues as they happen so I have a reliable reference for future maintenance, debugging, and updates.

---

## Troubleshooting Template
*This is the template I use for documenting bugs and issues I run into while working on the site.*

```markdown
## Error: [Exact error message or clear description of the issue]

### Context & Symptoms
* **Trigger Event:** [e.g., Happened right after activating Plugin X]
* **System Behavior:** [e.g., 500 Internal Server Error / blank white screen / layout breaking]

### Resolution Steps
1. [Step 1: e.g., Opened the file manager or server directory]
2. [Step 2: e.g., Isolated the issue by disabling or renaming the conflicting plugin]
3. [Step 3: e.g., Refreshed the dashboard to confirm the site was working again]

### Root Cause & Prevention
* [e.g., The plugin wasn’t compatible with the current WordPress version. Going forward, I’ll check compatibility before installing updates or new plugins.]
\```
